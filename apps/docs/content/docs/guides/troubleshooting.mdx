## Database Issues

If something seems off or you want a fresh start, delete the database file and relaunch Coodeen:

| Platform | Delete this file |
|----------|------------------|
| macOS | `~/Library/Application Support/coodeen/coodeen.db` |
| Windows | `%APPDATA%/coodeen/coodeen.db` |
| Linux | `~/.config/coodeen/coodeen.db` |

A new database will be created automatically on the next launch.

## AI Can't See Screenshots

Make sure you're using a vision-capable model:

- **OpenAI**: GPT-4o, GPT-4.1
- **Anthropic**: Claude Sonnet 4, Claude Opus 4
- **Google**: Gemini 2.5 Pro, Gemini 2.5 Flash

Models like GPT-4.1-nano don't support image inputs.

## Preview Not Loading

If the preview panel shows a blank page:

1. Make sure your dev server is actually running on the URL you entered
2. Check for CORS issues — some frameworks block iframe embedding
3. Try reloading the preview with the refresh button

## API Key Not Working

1. Verify the key is correct in Settings
2. Check that the key has the necessary permissions
3. Ensure you have billing set up with the provider
4. Try regenerating the key from the provider's dashboard

## Slow Responses

Response speed depends on your chosen model and network connection. Smaller models (GPT-4.1-mini, Claude Haiku) are faster. Larger models (GPT-4.1, Claude Opus 4) are more capable but slower.

## macOS Security Warning

Since the app is not yet code-signed, macOS may show a warning when you first open it. Right-click the app and select "Open" to bypass Gatekeeper, or go to System Settings > Privacy & Security and click "Open Anyway".
